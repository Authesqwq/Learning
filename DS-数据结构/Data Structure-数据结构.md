1.数据结构
- 数据：能输入到计算机中的符号。
	- 数据项：有独立含义的数据最小单位。
	- 数据元素：数据的基本单位。
	- 数据类型：数据的不可分割的单位。
- 数据结构：相互间存在关系的数据元素的集合。
$$DataStructure=(D,S)$$
- **逻辑结构**：数据元素间抽象化的相互关系（集合、线性结构、树形结构、图形结构）。
	- 线性结构：线性表（一般线性表、栈、队列、串、数组） “**一对一**”
	- 非线性结构：集合、树形结构“**一对多**”、图形结构或网状结构“**多对多**” 
- **存储结构**：存储关系（顺序存储、链式存储、散列存储、索引存储）。
	- **顺序存储**：逻辑上相邻的元素存储在物理位置也相邻的存储单元中
	- **链式存储**：借助指示元素存储地址的指针
	- 索引存储：索引表中的索引项（关键字+地址）
	- 散列存储：即哈希hash存储: $关键字 \xrightarrow{散列函数} 存储地址$

2.算法
- 算法：对步骤的描述。
- 特点：有穷性、确定性、可行性、输入(0或多)、输出（1或多）
- 目标：正确性、可读性、健壮性、效率与低存储量需求
- **时间复杂度**：
	- 频度: 语句在算法中被重复执行的次数 $$T(n)=O(f(n))$$
	- 加法规则：保留大的；乘法规则：相乘。

7.串：由0个或多个字符组成的有限序列

- 子串：任意个**连续**字符的子序列
- 字符在串中的位置：该字符在串中的序号；子串以第一个字符的位置来表示
- 空格串：全是空格；与”空串“区分
- 基本操作
	 ![[Pasted image 20260412184811.png]]
- 模式匹配：求子串在主串中的位置
	- 简单算法：从第一位开始一个一个比，一旦匹配失败，子串后移一位，直至匹配成功。$O(n*m)$
	- KMP算法：失败时主串指针不回退，仅将模式串指针回退到合适的位置，跳过必然失败的步骤。$O(n+m)$

	- 特殊矩阵：对称矩阵、上下三角矩阵、稀疏矩阵等

8.树：n个节点构成的有限集合。（层次结构；非线性结构）
	![[Pasted image 20260412190154.png]]
	- 子集不能相交；不能出现环
	- 递归定义：
		- n=0时，称为空树
		- n>0时：有且只有一个根结点，其余结点可分为m个互不相交的子集；每个子集也是树，成为根节点的子树
	- 术语：
		- 双亲（父）
		- 孩子（子）
		- 兄弟：有相同父节点
		- 祖先：到根节点路上的每个结点
		- 子孙：子树上的所有结点
		- 层度、高度、深度
			- 对于树：层数=高度=深度
			- 对于结点：深度为所在的层次；高度为以他为根的子树的层数
		- 度：
			- 结点的度：孩子的个数；若度为0，称为叶结点；若度大于0，称为分支节点
			- 树的度：树中结点的最大度数
		- 叉：分支数
			- 这是一个事先对树结构的一种约束；而度是对树此时状态的一种描述。
		- 序：
			- 有序树：各结点的子树从左到右有次序，不能互换
			- 无序树：相反
		- 路径：从上到下经过的结点序列
			- 长度：经过的边数
		- 森林：包含m棵互不相交的树的集合
- 性质：
	- 结点数n=边数+1=所有结点度数之和+1
	- m叉树的第$i$层最多有$m^{i-1}$个结点
	- 高度为$h$的$m$叉树最多有$\frac{m^h-1}{m-1}$个，最少有$h$个
	- 度为$m$、高度为$h$的树最少有$h-1+m$个结点
	- 具有$n$个结点的$m$叉树（度为$m$的树）的最小高度$h=\lceil \log_{m}{(n(m-1)+1) \rceil}$
	- 具有$n$个结点的$m$叉树的最大高度为$n$; 度为$m$的最大高度为$n-m+1$
- 二叉树：n个结点构成的有限集合，度只能为0,1,2
	- 递归定义
	- 严格区分左右子树
	![[Pasted image 20260412192857.png]]
	- 特殊二叉树
		- 满二叉树：度=0或2
		- 完全二叉树：必须从左到右连续排列
			- 从上到下，从左到右依次编号$1,2,...,n$
			- 对于$i$号结点：左孩子$2i$;右孩子$2i+1$;父亲$\lfloor \frac{i}{2} \rfloor$
			- 最后一个分支结点的编号为$\lfloor \frac{n}{2} \rfloor$，它与它前面的为分支结点，后面的都是叶结点
			- 最多只可能有一个度为1的结点，n为奇数时，没有；n为偶数时，有一个，编号为$\lfloor \frac{n}{2} \rfloor$，且只有左孩子没有右孩子
			- 叶结点只可能在最后两层出现
			- 性质：
				- 具有n个结点的完全二叉树的高度:$$2^{h-1}\le n \le 2^h$$此时取对数求得$$h=\lceil log_{2}{(n+1)} \rceil$$
	- 性质：
		- 叶结点数=双分支结点数+1
		- 二叉树的第$i$层最多有$2^{i-1}$个结点，高度为$h$的二叉树共最多有$2^h-1$个
	- 存储结构：
		- 顺序存储：完全二叉树最合适
			- 完全二叉树：将编号作为一维数组的下标，从1开始存储
			- 一般二叉树：增加虚拟的占位结点，将其视作完全二叉树
			- 右单支树：最坏情况，浪费指数级的存储空间
		- 链式存储：lchild,data,rchild
			![[Pasted image 20260412195021.png]]
			- 空指针的个数=结点数+1
			- 三叉链表：增加一个指针域用来指向父亲
				![[Pasted image 20260412195300.png]]
	- 二叉树遍历：
		- 前序遍历：根结点$\to$递归遍历左子树$\to$右子树
		- 中序遍历：递归遍历左子树$\to$根结点$\to$右子树
		- 后序遍历：递归遍历左子树$\to$右子树$\to$根结点
		- 层序遍历：从上到下，从左到右
	- 二叉树构造：给定序列构造出的树不唯一
		- 前序、后序、层序遍历能确定根结点，中序不确定，但知道根节点可以推出左右
	- 线索二叉树：
		- 线索：利用空指针指向前驱和后继
		- 改造后称作X序线索二叉树，过程称为线索化
		- 单个节点的结构：lchild,ltag,data,rtag,rchild（tag=1为线索，tag=0为指针）
		- 中序线索树：
			- 找中序前驱：若ltag=1，即左指针所指；若ltag=0，则为左子树最右下结点
			- 找中序后继：右指针；右子树最左下
			- 此时遍历为从第一个结点开始不断找后继的过程，省去了栈空间
		- 先序线索树：
			- 前驱：无法有效寻找
			- 后继：rtag=0时，若有左孩子，则后继为左孩子；若无，则后继为右孩子（必有）
			- 遍历：从根节点不断找后继，省去了栈空间
		- 后续线索树：
			- 前驱：rtag=0时，若有右孩子，则为前驱；若无，则前驱为左孩子（必有）
			- 后继：无法有效寻找
			- 遍历：仍需栈的支持
	- 树的存储
		- 双亲表示法：data,parent（存父亲的下标可以索引）
			- 找父亲很容易；找孩子很麻烦，需要遍历
		- 孩子表示法：data,firstchild（链表）
			 ![[Pasted image 20260412202800.png]]
			- 找孩子很容易；找父亲很麻烦，也要遍历（除非改变定义，增加一个parent）
		- 孩子兄弟表示法：firstchild,nextsibling（二叉链表）
			- 实际上是用二叉树存储一般树
			
			![[Pasted image 20260412203039.png]]
	- 转化二叉树：孩子兄弟表示法
		- 一般树转化后，根节点的右子树一定为空
		- 森林转化后，可能有右子树
			- 把森林中的每棵树转化完后，从左到右依次作为右孩子连接
			![[Pasted image 20260412203635.png]]
			- 转换回森林同理
	- 树的遍历
		- 先根遍历：先访问根，再依次遍历每个子树
			- 原树的先根遍历相当于转化后的二叉树的先序遍历
		- 后根遍历：先遍历每个子树，再访问根
			- 即中序遍历
	- 森林的遍历
		- 先序遍历：同理
		- （中序）后序遍历：同理
	- 哈夫曼树
		- 编码：
			- 二进制编码：前缀歧义问题
			- 等长编码：无歧义，但不是最短方案
		- 哈夫曼编码：根据出现的次数，不断合并最小的两个，构造一棵二叉树，即哈夫曼树
		- 规则：左边记为0，右边记为1，此时编码为其路径。每一个词都成为叶结点，因此不可能出现前缀歧义问题
		- 编码长度：叶结点频率×路径长度的和
	- 并查集：高效地处理集合的合并与查询
		- 把集合用树（森林）来表示
		- 查询：寻找根结点
